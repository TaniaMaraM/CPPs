# PhoneBook - Cenários de Teste

## Teste 1 - ADD básico
> ADD
First Name: John
Last Name: Doe
Nickname: JD
Phone Number: 123456789
Darkest Secret: Afraid of cats
Contact added successfully!

## Teste 2 - SEARCH com PhoneBook vazio
> SEARCH
Error: PhoneBook is empty!

## Teste 3 - ADD com campo vazio (deve pedir de novo)
> ADD
First Name: [Enter vazio]
Error: Field cannot be empty!
First Name: Jane

## Teste 4 - SEARCH após adicionar
> ADD
[preencher contacto]
> SEARCH
[tabela aparece]
Enter index: 0
[mostra detalhes]

## Teste 5 - SEARCH com índice inválido
> SEARCH
Enter index: 99
Error: Index out of range!

## Teste 6 - SEARCH com input não-numérico
> SEARCH
Enter index: abc
Error: Invalid input!

## Teste 7 - Adicionar 9 contactos (deve substituir o 1º)
> ADD [contacto 1]
> ADD [contacto 2]
...
> ADD [contacto 9] (substitui contacto 1)

## Teste 8 - Campo com mais de 10 chars (deve truncar na tabela)
First Name: Christopher
Last Name: Montgomery
Nickname: ChrisMont
[Na tabela aparece: Christoph., Montgome., ChrisMon.]

## Teste 9 - EOF (Ctrl+D)
> ADD
First Name: [Ctrl+D]
Input cancelled.

## Teste 10 - Comando inválido
> HELP
Error: Invalid command! Use: ADD, SEARCH, EXIT

## Teste 11 - EXIT
> EXIT
Goodbye!
