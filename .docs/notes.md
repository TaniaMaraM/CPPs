Main concept: Virtual functions and runtime polymorphism
What you need to understand:
Virtual functions allow derived classes to override base class behaviour
Without virtual, the compiler calls the method based on the pointer type (compile-time)
With virtual, it calls based on the actual object type (runtime)

Key concept: virtual keyword tells compiler "don't decide which function to call at compile time - wait until runtime and check the actual object type."
